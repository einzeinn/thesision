# RFC-008

## Canonical Export Pipeline

### Objective

JSON is the single source of truth. It remains the complete, portable, and
auditable reasoning record. Markdown is a concise report for people.

### Requirements

- JSON export must not be changed, summarized, or depend on a model.
- Markdown export reads the canonical JSON export as its only input.
- Markdown does not read runtime state or UI state.
- Markdown may use the configured model through the Orchestrator to compress
  the JSON into the question, conclusion, confidence, key evidence,
  trade-offs, caveats, and next steps.
- The model must not invent claims, sources, or recommendations; every report
  statement must be traceable to the JSON input.
- If the provider is unavailable or returns an invalid result, a deterministic
  renderer from the same JSON provides a concise fallback so export remains
  available.

### Definition of Done

```text
Persisted session
      ↓
Canonical JSON export
      ↓
Concise Markdown report
```

The JSON export remains identical to the persisted session. Markdown is
readable and concise without adding information that is absent from JSON.
