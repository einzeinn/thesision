# RFC-013 — Constellation-Based Reasoning Graph Layout

## Objective

Replace the serpentine placement with a template-driven constellation layout.
This changes only frontend node positioning and rendering.

## Layout Engine

The renderer selects one of 30 geometric constellation templates from a stable
hash of the canonical question. The hash is deliberate: it gives distinct
questions different layouts while making replay, imported sessions, polling,
and continuation stable. `Math.random()` is not used.

Each debate round occupies an appended local constellation region. Existing
round positions depend only on their round, node type, and stable template, so
new rounds never reflow prior regions. The Question is first; Conclusion is
isolated after the final Judge.

Templates define approximate primary-node coordinates. A minimal deterministic
Y-only collision relaxation can prevent overlap; it never changes a node's
template X coordinate. Evidence satellites use radial clusters, Perspective
satellites use a loose fan, and no decorative data is added.

## Constraints

- Do not change backend, API, reasoning semantics, lifecycle, replay order,
  progressive reveal timing, or export schema.
- Keep the existing sticky side panels and keyboard interactions.
