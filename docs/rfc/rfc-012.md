# RFC-012 — Explainable Reasoning Reports

## Objective

Improve Markdown reasoning reports using only the canonical exported session.
JSON remains the portable source of truth and is not changed.

## Report Content

Reports include:

- an explainable confidence breakdown using only values stored by the confidence
  evaluator; unavailable dimensions are explicitly `Unknown`;
- classified evidence with summary, source, supports, confidence, and strength;
- an observable reasoning trace (not chain-of-thought);
- a concise recommendation, rationale, and conditions for reconsideration;
- caveats rendered as uncertainty, impact, and resolving evidence, using
  `Unknown` when canonical state does not provide a field;
- decision-change conditions derived from recorded assumptions, unknowns,
  unresolved conflicts, and next steps.

## Constraints

- Do not change orchestration, graph/session data, evidence retrieval, JSON
  exports, replay, or lifecycle behavior.
- Do not infer citations, URLs, statistics, confidence dimensions, or hidden
  reasoning.
- AI-composed Markdown must satisfy the required visible section structure;
  otherwise use the deterministic canonical-session renderer.
