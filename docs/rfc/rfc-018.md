# RFC-018 — Scannable Confidence Signals in Markdown

## Objective

Make the persisted confidence score readable at a glance in Markdown reports
without inventing a component-score breakdown.

## Presentation

The report renders the canonical overall score as a ten-cell visual bar and a
percentage. It then renders three compact reason signals:

- Evidence quality — pass when the recorded average is at least 50/100;
- Unresolved conflicts — pass only when the recorded count is zero;
- Perspective coverage — pass when at least three perspectives were recorded.

Each signal includes its stored value. The pass/fail markers are reading aids,
not confidence components, and they do not imply that the values add up to the
overall score. Missing values render as unavailable rather than as a pass.

## Constraints

- Use only canonical confidence fields already persisted by the evaluator.
- Do not modify the confidence formula, session state, JSON export, or
  reasoning lifecycle.
