# RFC-023 — Adaptive Refinement Targets for Later Rounds

## Status

Proposed

## Objective

Make reasoning rounds after Round 1 investigate the most useful unresolved
artifact from the prior round, rather than producing a generic new set of
hypotheses from the original question and Judge summary alone.

The goal is more useful, non-repetitive insight during multi-round and
Continue Reasoning sessions while retaining an understandable, inspectable
reasoning pipeline.

## Decision

Adopt **adaptive focus, fixed pipeline**.

The pipeline order remains:

Question → Hypothesis → Evidence → Perspectives → Judge → Confidence → Conclusion

Later rounds still begin with the Hypothesis stage. However, that stage must
be given one explicit refinement target selected from canonical artifacts of
the immediately preceding round. The target may originate from Evidence,
Perspectives, Conflict, or Judge output.

Conclusion is never a refinement source. It remains the final recommendation
for the completed session and must not anchor later investigation.

## Refinement Target Priority

For Round 2 and later, select one target using this priority order:

1. An unresolved Conflict, because it represents a material decision gap.
2. A missing, weak, or explicitly unverified Evidence item.
3. A meaningful disagreement between recorded Perspectives.
4. A Judge decision condition, unknown, or requested validation.
5. The latest Judge synthesis only when no more specific artifact exists.

The selection must use canonical session artifacts only. It must not invent a
gap merely to make a later round appear different.

## Required Later-Round Behavior

The selected refinement target is supplied to the Hypothesis and Evidence
agents alongside the question and existing bounded prior-round context.

Later-round hypotheses must do one of the following:

- test a disputed claim;
- verify a weak or missing evidence gap;
- quantify a trade-off between perspectives; or
- establish a decision threshold or condition.

Examples:

- Conflict: “At what deployment-failure rate does Docker Compose no longer
  meet the startup's reliability requirement?”
- Evidence gap: “Does self-hosting become cheaper after GPU utilization,
  engineering time, and on-call costs are included?”
- Perspective disagreement: “Does the target latency budget justify the
  operational complexity of self-hosted inference?”
- Decision condition: “Can a managed container platform satisfy the required
  GPU, storage, and rollback requirements without Kubernetes?”

Evidence retrieval in later rounds must search specifically for the selected
target and continue to exclude previously seen source URLs.

## Explainability and Graph Semantics

The graph must preserve the existing node types and stage ordering. A
later-round Hypothesis may record which prior artifact triggered it, but it
does not change into an Evidence, Perspective, Conflict, or Judge node.

If trigger metadata is rendered, it must identify the artifact type and
canonical node or item identifier. It must not expose hidden chain-of-thought.

## Confidence Semantics

This RFC does not recalibrate the confidence formula by itself.

When implemented, confidence must continue to use cumulative unique evidence
and perspectives, while unresolved-conflict penalties come only from the
latest Judge. A future calibration change may award resolution only when the
latest Judge explicitly records that a prior conflict was resolved; it must
not infer resolution from the existence of another round.

## Non-Goals

This RFC does not:

- alter the backend API or canonical JSON schema without a separate RFC;
- allow stages to execute in a different order;
- use the final Conclusion as a continuation prompt;
- fabricate a new issue when all recorded artifacts are resolved; or
- change replay behavior, export format, or graph layout.

## Risks and Mitigations

- **Overly narrow later rounds:** retain the original question and bounded
  Judge context alongside the target.
- **Incorrect target selection:** use deterministic priority rules and show
  the selected artifact in inspectable session data.
- **False confidence improvement:** require an explicit latest-Judge
  resolution record before any future score adjustment.
- **Graph complexity:** keep the pipeline fixed and make trigger metadata
  optional presentation detail rather than a new graph node type.

## Acceptance Criteria

- A Round 2+ hypothesis explicitly addresses a canonical unresolved conflict,
  evidence gap, perspective disagreement, decision condition, or Judge
  synthesis in that priority order.
- Later-round evidence is relevant to that target and does not repeat a prior
  URL unless no new verified source is available.
- The graph continues to display the same stage order and node semantics.
- Exported JSON remains portable and backward compatible.
- A later round does not receive the final Conclusion as its refinement source.
- Confidence does not increase merely because another round was run.
