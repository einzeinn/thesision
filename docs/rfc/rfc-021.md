# RFC-021 — Cumulative Debate Rounds

## Objective

Make later reasoning rounds and continued sessions investigate unresolved
decision gaps instead of repeating the original question and evidence search.

## Round Context

Round 1 receives the user's question and optional engineering context. Every
later round receives an internal refinement context composed from canonical
artifacts already stored in the session:

- the latest Judge synthesis, validated claims, conflicts, and unresolved
  conflicts;
- the latest confidence signals; and
- a bounded history of previously considered evidence claims, titles, URLs,
  and quality labels from graph nodes.

This context is an agent input only. It does not alter the public API or the
canonical JSON export schema.

## Agent Behavior

- Hypothesis R2+ must refine, test, or challenge unresolved gaps from the
  prior Judge rather than restating unchanged hypotheses.
- Evidence R2+ must seek materially new sources relevant to those gaps and
  avoid URLs already present in the supplied evidence history.
- Continue Reasoning uses the retained parent graph nodes as the same context,
  so an imported or continued session remains cumulative.

## Constraints

- Round 1 behavior remains unchanged.
- Do not fabricate citations or suppress an explicit no-source result.
- Do not change the reasoning graph schema, API contracts, replay, export, or
  frontend behavior.
