# RFC-020 — Incremental Graph Rendering and Soft Camera Follow

## Objective

Preserve visual continuity while a reasoning session grows. Existing graph
artifacts must not blink or re-enter when a later stage becomes available.

## Incremental Rendering

The renderer retains existing SVG nodes and edges when polling adds artifacts.
Only new nodes and their new edges receive the existing entrance and line-draw
animations. Explicit replay and imported-session rendering may reset the graph
intentionally.

## Soft Camera Follow

The graph viewport persists across incremental updates. When a newly revealed
primary node lies outside a central safe area, the viewport pans smoothly to
bring it into view. It does not follow every satellite node and it does not
move when the new primary node is already visible.

Manual pan or zoom suspends auto-follow temporarily. Users retain full pan and
zoom control at all times. Reduced motion disables camera animation.

## Constraints

- Do not change node coordinates, graph data, reasoning order, replay order,
  backend behavior, or API/session contracts.
- Keep the existing node and edge entrance timing; this RFC only controls
  whether existing DOM artifacts are recreated and whether the viewport moves.
