# RFC-022 — Cumulative Evidence Reporting

## Objective

Make the human-readable Markdown report reflect the cumulative evidence
already stored by multi-round and continued reasoning sessions.

## Rules

- JSON remains the complete, portable canonical session record; no schema,
  API, agent workflow, or evidence retrieval behavior changes.
- Markdown gathers evidence from every completed `evidence` graph node,
  deduplicates it by canonical URL (or normalized claim when no URL exists),
  and identifies the total unique-source and evidence-round counts.
- The compact table shows at most eight of the most recent unique sources.
  Earlier sources remain available in JSON rather than being silently treated
  as if they were not considered.
- Confidence evidence quality and perspective coverage use the same cumulative
  graph artifacts. Repeated perspective labels count once for coverage.
- Unresolved conflicts remain sourced only from the latest Judge result. A
  historical conflict must not keep reducing confidence after a later Judge
  resolves it.

## Constraints

- The existing confidence formula and score weights do not change.
- This is a reporting and aggregation correction, not a score calibration.
- Imported sessions without historical graph nodes fall back to their current
  state fields for backwards compatibility.
