# RFC-009 — Graph Layout Redesign

## Objective

Replace the radial force-directed graph with a constrained, horizontally
progressing DAG. The layout communicates reasoning order first and retains
small organic variation only within a stage.

## Timeline Layout

Question is the root anchor. Each stage occurrence owns a fixed X coordinate:

```text
Question → H1 → E1 → P1 → C1 → J1 → H2 → E2 → P2 → C2 → J2 → Conclusion
```

This stage-per-round timeline is intentional: assigning one fixed column to
every node type would make `Judge R1 → Hypothesis R2` travel backward across
the graph. Every normal flow and continuation edge must instead progress to the
right.

Primary nodes use fixed X positions. Y positions are computed from each
column's sibling order, parent position, collision spacing, and a deterministic
small organic offset. Satellite nodes remain close to their parent.

## Constraints

- No backend, API, reasoning-pipeline, node-data, animation-timing, or replay
  changes.
- Physics does not determine hierarchy. It may only resolve small collisions
  after initial positioning, and the graph settles before paint.
- Edges use smooth curves. Avoid crossings and long vertical paths.
- The Question remains the left root anchor; Conclusion remains the right end.
- Side panels remain fixed while their own content can scroll. The graph canvas
  owns the remaining center workspace.

## Success Criteria

The graph immediately reads as Question → Hypothesis → Evidence → Perspective
→ Conflict → Judge → Conclusion, while its vertical placement remains lightly
organic rather than perfectly grid-aligned.
