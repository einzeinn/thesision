# RFC-019 — Responsive Reasoning Workspace

## Objective

Make the existing reasoning workspace usable on narrow mobile screens without
changing the reasoning pipeline, session lifecycle, graph data, animations, or
export behavior.

## Layout

Desktop remains the primary presentation. At narrow viewport widths, the
three-panel workspace becomes a single vertical flow:

1. the input sidebar occupies the top of the page and remains expandable;
2. the graph remains the central, fixed-height interaction surface;
3. session evidence and output controls appear below the graph.

The page must not retain a desktop minimum width or force horizontal document
scrolling. The graph itself continues to support its existing pan and zoom
interaction within its own viewport.

## Mobile Rules

- Controls use full-width touch targets and remain readable at 320px wide.
- The graph header may wrap; its legend may scroll horizontally instead of
  shrinking node-type labels into illegibility.
- Sticky desktop side panels become normal-flow panels on mobile.
- The existing sidebar toggle remains available, but a collapsed mobile sidebar
  must not leave an unusable 64px-wide input area.

## Constraints

- Do not change backend behavior, API contracts, reasoning data, replay, or
  export semantics.
- Preserve the line-art visual language, palette, and reduced-motion support.
