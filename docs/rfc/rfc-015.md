# RFC-015 — Concise Canonical Reasoning Reports

## Objective

Make Markdown reports decision-readable rather than an exhaustive rendering of
session artifacts.

## Rules

- Present the persisted overall confidence score with recorded signals, but do
  not reconstruct component scores unless those scores are persisted; displayed
  signals must not imply that they sum to the overall score.
- Render evidence in a compact table. Deduplicate identical sources while
  preserving every source link and source focus. RFC-022 defines the
  cumulative multi-round aggregation and eight-row display limit.
- Limit trade-offs and caveats to the most relevant recorded items, using
  concise excerpts rather than full analysis paragraphs.
- Show at most eight key decision-change conditions in an expandable block,
  prioritizing recorded unknowns, conflicts, and planned validation.
- Resolve `Why` from conclusion rationale, then the latest Judge synthesis,
  then validated claims. Do not show `Unknown` when a canonical fallback is
  available.

The complete canonical JSON export remains the exhaustive source of truth.
