# RFC-001 — Transform Summarization into Argumentative Reasoning

## Objective

Upgrade the reasoning engine so Thesision performs structured engineering reasoning instead of sequential summarization.

## Problem

Current pipeline mainly summarizes hypotheses and perspectives.

The Judge produces summaries rather than evaluating competing arguments.

Confidence changes are weakly justified.

The reasoning graph does not clearly show why a conclusion was reached.

## Requirements

### Hypothesis

- Generate 2–5 competing hypotheses.
- Each hypothesis must have a stable ID.
- Each hypothesis starts with confidence = 0.

### Evidence

Each evidence item must contain

- id
- source
- summary
- supports hypothesis
- contradicts hypothesis
- evidence strength
- reliability score

### Evidence Scoring

Every evidence updates hypothesis confidence.

Example

+15
-8
+4

instead of qualitative labels only.

### Conflict Detection

Detect

- contradictory evidence
- unsupported claims
- duplicated reasoning

Generate Conflict objects.

### Debate

Before Judge executes

each hypothesis must answer

- strongest supporting evidence
- strongest opposing evidence
- remaining uncertainty

### Judge

Judge must compare hypotheses.

Not summarize.

Judge must explain

WHY hypothesis A wins.

WHY hypothesis B loses.

WHY confidence changed.

### Confidence

Confidence must be computed from

- evidence quality
- evidence agreement
- unresolved conflicts
- iteration history

### Conclusion

Conclusion must reference

winning hypothesis

supporting evidence

remaining risks

recommended decision

## Non Goals

No frontend redesign.

No export redesign.

No API breaking changes.

## Definition of Done

- Judge no longer produces summaries.
- Evidence updates confidence.
- Debate stage exists.
- Conflict detection exists.
- Existing tests pass.
