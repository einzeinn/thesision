# RFC-011 — Serpentine Reasoning Graph

## Objective

Keep the constrained reasoning DAG readable as debate rounds increase without
expanding its width by five columns per round.

## Layout

The graph uses a serpentine timeline:

```text
Question → H1 → E1 → P1 → C1 → J1
                              ↓
J2 ← C2 ← P2 ← E2 ← H2
↓
H3 → E3 → P3 → C3 → J3
```

Each round occupies a distinct vertical row. The next Hypothesis is directly
below the prior round's Judge; direction alternates by round. X still conveys
the local stage order, while the row and continuation edge convey debate
progress. Conclusion sits below the final Judge.

Stage-specific vertical offsets remain within each row, satellite clusters
remain radial and data-driven, and all placement is deterministic.

## Evidence Panel

The session evidence panel presents each claim as a compact preview of at most
two sentences, followed by its source link. Full source content remains
available through that link and through the graph-node popup.

## Constraints

- Do not change reasoning, API/backend contracts, session data, replay,
  progressive reveal, animation timing, or exports.
