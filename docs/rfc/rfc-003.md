# RFC-003 — Demo-first UI Experience

## Objective

Optimize the interface for live demonstrations.

## Requirements

Reasoning must animate stage-by-stage.

Each stage

Question

↓

Hypothesis

↓

Evidence

↓

Conflict

↓

Judge

↓

Conclusion

appears progressively.

Current stage highlighted.

Completed stage locked.

Node colors reflect status

running

completed

conflict

warning

failed

Confidence updates live.

Conclusion appears only after pipeline completes.

Export disabled while reasoning is active.

Loading skeletons replace empty placeholders.

Transitions remain under 300ms.

Reduced motion remains supported.

## Non Goals

No visual redesign.

No framework migration.

## Definition of Done

Viewer understands reasoning progress within five seconds.
