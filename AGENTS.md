# Thesision

> Reason Before Decision.

## Before You Start

Before writing any code:

1. Read every document inside the `docs/` directory.
2. Follow every rule defined in `docs/AIInstructions.md`.
3. Treat the documentation as the single source of truth.
4. Prioritize the MVP over Future Features.
5. Never redesign the architecture without updating the documentation first.

---

## Project Identity

Thesision is NOT a chatbot.

Thesision is an AI reasoning platform focused on transparent, evidence-driven reasoning for software engineers.

Every implementation should reinforce this identity.

---

## Development Principles

- Read first, code second.
- Keep components modular.
- Do not guess requirements.
- If documentation and implementation conflict, update the documentation before changing the code.
- Never implement Future Features unless explicitly requested.

---

## Final Reminder

When uncertain:

Stop.

Read the documentation again.

Then continue.
